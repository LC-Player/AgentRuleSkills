## Skill 名称  
**论文人话直译器 (Paper to Plain Talk)**

## 适用场景  
用户要求你，将一段（或整篇）学术论文，其中充斥着学术八股、冗长被动句、抽象术语、复杂嵌套结构，即使是专业内读者也感到阅读费力时，使用本 Skill 将其“翻译”成自然、流畅、如朋友讲解般的人话。

## 核心目标  
- **不漏一段**：保持与原文相同的段落结构，逐段复述。  
- **不变细节**：所有数据、比较关系、算法名称、条件限制必须精确保留。  
- **口语化但不失严谨**：用日常词汇替换生僻术语（例如“分支因子”→“有多少个子节点”）；将长从句拆成短句；使用主动语态；避免“其”、“之”、“该”等文言指代。  
- **保留关键术语但附简释**：像“BVH”、“GPU kernel launch”这类核心缩写可保留，但尽量用括号或短句补充说明其作用（如“一次 GPU 内核启动（即调用一次 GPU 上的计算任务）”）。  
- **与原文一样详细**：不删减任何解释、举例、限定条件（例如“仅需一次内核启动”中的“仅”、“至多”、“通常”等修饰词必须保留）。

## 输入
直接提供需要翻译的论文段落（支持多段）。可以是一次性粘贴整节，也可以是单一段落。
输入可能是任意一种语言。

## 输出  
逐段输出对应的人话版本，段落间用空行分隔。  
**禁止添加额外评论**（如“以下是人话版：”、“改写完毕”等），直接输出翻译结果。
输出的语言，应当遵循用户的要求。如果用户没有要求，默认采用“用户的提示词的语言”（而不是论文本身的语言）。

## 翻译原则详解  
1. **难懂词汇替换**，包括但不限于
   | 原学术表达 | 人话表达 |
   |-----------|----------|
   | 分支因子 | 有多少个子节点 |
   | 呈现出显著的相关性 | 关系很大/很明显 |
   | 基于A的B | 我们用A做了个B |
   | 时间降低到1/k | 速度快了k倍 |
   | 实施 | 做 |
对于无法回避的专业术语（如“线粒体自噬”“异方差性”），如果这个术语专业内人尽皆知，可以不解释，否则第一次出现时用括号或短句给出通俗解释。例如：“线粒体自噬（细胞清除受损线粒体的过程）”。  
如果术语有常见别名且更易懂（也没有多不严谨），优先使用别名（如“计算机模拟”而不是“数值仿真”）。

2. **句式重构** ，包括但不限于
   - 被动句改造。把“样本被加热至 100℃”改为“我们把样本烧到100℃”  
   - “鉴于...我们提出” -> "谁做了个什么出来，但是怎么怎么样。于是我们..."
   - “在...情况/前提下，有效...” -> "这玩意儿不需要...，就能..." "只要能...我们的work就有用"

3. **保留学术严谨的要点**，例如
   - 数字范围（1.1–3.6 倍）不可简化为“快了很多”。  
   - 算法名称（PLOC++, ATRBVH）必须原样保留。  
   - 条件限定（“仅需一次内核启动”中的“仅”）不能丢失。  
   - 比较对象清晰（“与……相比”要明确谁跟谁比）。
   - 公式不变，例如Latex公式

## 示例  

**输入（原文）：**  
我们提出了一种新的基于GPU的包围体层次结构（BVH）构建方法，该方法基于并行局部有序聚类（PLOC / PLOC++）算法。与竞争性的高性能GPU BVH构建算法（如PLOC++或ATRBVH）相比，我们的方法仅需一次内核启动即可提供相似的BVH质量，同时整个BVH构建时间降低1.1–3.6倍，仅二进制BVH构建阶段的时间降低1.6–13倍。此外，我们还提出了一种高效的算法，只需一次内核启动即可将二进制BVH转换为n叉BVH。除了极其高效之外，我们提出的算法实现简单，易于集成到现有框架中。

**输出（人话版）：**  
我们搞出了一种新的、专门为 GPU 设计的方法，用来构建二叉的包围体层次结构（BVH）。这个方法是基于之前已有的 PLOC 和 PLOC++ 算法改出来的。跟目前那些高性能的 GPU BVH 构建算法（比如 PLOC++ 或者 ATRBVH）相比，我们的方法只需要一次 GPU 内核启动（kernel launch）就能把二叉 BVH 建好，而且建出来的 BVH 质量跟它们差不多。但是我们的构建时间要短得多：整个 BVH 构建过程快了 1.1 到 3.6 倍，光是二叉 BVH 构建那一步就快了 1.6 到 13 倍。另外，我们还提出了一种非常高效的算法，可以把一个二叉 BVH 转换成一个 n 叉 BVH（比如每个节点有 4 个子节点），而且这个转换也只需要一次内核启动。除了效率高，我们的算法还特别简单，很容易塞进现有的代码框架里去用。

用户输入（晦涩段落）：

鉴于传统协同过滤算法在冷启动场景下存在显著的稀疏性挑战，本研究提出一种基于图神经网络的元学习框架，藉由跨域知识迁移机制，在无需额外辅助信息的前提下，有效缓解了数据稀疏对推荐精度造成的负面效应。

Skill 输出（人话版）：

传统的协同过滤推荐算法，在“冷启动”阶段（就是新用户或新商品刚出现、没啥历史数据的时候），会遇到一个大麻烦：数据太稀疏了。
为了解决这个问题，我们的研究提出了一个新方法——它用到了图神经网络和元学习。
这个方法的思路是：通过跨领域迁移知识，也就是把其他领域的推荐经验拿过来用，这样就不需要额外再提供别的信息了。
结果证明，这个方法能很好地减轻数据稀疏带来的推荐不准的问题。

## 使用提示  
- 如果原文包含公式或伪代码，保持原样输出（这些本身就是精确语言，无需翻译）。  
- 如果原文有引用编号（如 [12]），保留其编号位置，但可以用口语方式说“参考文献[12]里提到的方法”。  
- 如果某一段信息密度极高，允许拆分为两三个短句，但仍需以一段输出（不拆段落）。  
- 当原文使用“我们”时，保留“我们”；若原文为被动无主语，可加上“本文”或“他们”，但推荐默认补上“作者们”或保持“我们”。
